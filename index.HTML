Após adicionarmos um novo arquivo no repositório, será necessário rea- lizar um commit, que é uma ação que registra uma mudança no códi- go do seu repositório. Quando fazemos um commit, estamos salvando uma versão do projeto com as mudanças que fizemos. Cada commit possui um identificador único que permite rastrearmos e gerenciarmos as mudanças que foram feitas no repositório ao longo do tempo. Ao fa- zer um isso no GitHub, é importante incluir uma mensagem descritiva que explique o que foi alterado e por quê. Isso ajuda a manter o reposi- tório organizado e facilita a compreensão das mudanças que foram fei- tas no projeto. Após preencher a mensagem descritiva, basta clicarmos em Commit new file para criar um novo commit.
